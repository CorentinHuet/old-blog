Les bonnes pratiques pour le developpement.

Pourquoi utiliser des normes ?

Il est important d'établir et d'utiliser des normes de codage notamment lorsqu'on travaille en équipe, mais il est quand même utile de les mettre en place lorsqu'on travaille seul.

Voici les règles essentielles :

1. L'organisation des fichiers : Il est important, dès le commencement d’un projet, de bien réfléchir et définir une arborescence propre. C’est-à-dire, une organisation des fichiers qui soit adaptée au projet, au langage et/ou aux frameworks utilisés.

Typiquement, lorsqu’on débute dans le développement, on a créé souvent tous les fichiers dans le même répertoire. Or, il est important de bien séparer les fichiers qui ont des utilités différentes.

2. De nommage des fichiers : Souvent, cette règle dépend des entreprises et des conventions définies dans celles-ci. Mais, un des principes qu’on retrouve, c’est de nommer ces fichiers du même nom qu’on y nomme ce qui est défini.

3. De nommage des classes, variables, fonctions, etc : Premièrement assurez-vous que les noms de ces éléments sont intuitifs. Il faut utiliser des caractères latins (0-9, a-z, A-Z), pour nommer une constante on l'écrit uniquement en majuscules et on met un underscore pour séparer les mots, pour nommer une fonction ou une variable on utilise du camelCase on commence par une minuscule puis on utilise une majuscule pour écrire un autre mot (exemple : monAge) Attention de ne pas nommer vos éléments avec mot réservé au langage.

4. L'indentation du code : L’indentation consiste à ajouter des tabulations ou des espaces à ses lignes de code. Il est important d'avoir un code bien indenté il est essentiel à sa bonne lecture et maintenabilité.

5. Les commentaires : Les commentaires ont une importance primordiale, ils sont essentiels à la bonne compréhension d'un morceau de code. Attention à ne pas tomber dans l'excès, et commenter chaque ligne de code. Commenter son code sera non seulement utile pour les personnes qui regarderont votre code mais aussi pour vous-même quand vous voudrez reprendre votre code après plusieurs semaines sans avoir travaillé dessus.

Optez pour la documentation

La documentation de code est souvent négligée, mais elle est d'une importance capitale. Écrivez des commentaires clairs et concis pour expliquer la logique complexe, les fonctions, les classes et les API. Une documentation bien entretenue facilite la collaboration et la maintenance de l'application au fil du temps.

Gardez un oeil sur la sécurité.

La sécurité est une préoccupation majeure dans le développement logiciel. Assurez-vous de connaître les meilleures pratiques en matière de sécurité et protégez votre application contre les vulnérabilités courantes, telles que les injections SQL, les failles XSS et CSRF. Effectuez des audits de sécurité réguliers pour identifier et résoudre les problèmes potentiels.
